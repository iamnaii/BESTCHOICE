**W1 вЂ” `as any` cast for Decimal arithmetic (overdue.service.ts) вЂ” CRITICAL SMELL**
```typescript
remainingAmount: (p.amountDue as any).sub(p.amountPaid as any),
```
`p.amountDue` and `p.amountPaid` are `Prisma.Decimal` at runtime (because they are `@db.Decimal(12,2)` in the schema), so `.sub()` will work. However, the `as any` cast defeats TypeScript's type safety вЂ” if the Prisma type changes or the query uses a different field, this will fail silently at runtime.  
**Fix:**
```typescript
remainingAmount: new Prisma.Decimal(String(p.amountDue)).sub(new Prisma.Decimal(String(p.amountPaid))),
```
Or cast properly: `(p.amountDue as Prisma.Decimal).sub(p.amountPaid as Prisma.Decimal)`
