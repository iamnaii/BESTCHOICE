#### 🔴 Critical — Missing `deletedAt: null` in Sale queries (accounting.service.ts)

**File:** `apps/api/src/modules/accounting/accounting.service.ts`  
**Lines:** ~364, ~383 (two new `sale.findMany` calls inside the annual-P&L function)

```typescript
// ❌ Missing deletedAt: null
this.prisma.sale.findMany({
  where: { createdAt: dateRange, ...branchFilter },
  select: { saleType: true, netAmount: true, downPaymentAmount: true, createdAt: true },
})

// ❌ Missing deletedAt: null
this.prisma.sale.findMany({
  where: { createdAt: dateRange, ...branchFilter },
  select: { createdAt: true, product: { select: { costPrice: true } } },
})
```

The `Sale` model has `deletedAt DateTime?` (confirmed in `schema.prisma`). Soft-deleted sale records will be included in annual P&L revenue and COGS calculations, corrupting financial reports.

**Compare:** Line ~143 in the same file correctly uses `deletedAt: null`:
```typescript

  where: { createdAt: dateRange, deletedAt: null, ...branchFilter },
  ...
})
```

**Fix:** Add `deletedAt: null` to `where` clauses of both affected queries.

#### ⚠️ Warning — `Number()` on Decimal money fields in contracts.service.ts

**File:** `apps/api/src/modules/contracts/contracts.service.ts`  
**Lines:** 174–177, 563–566

```typescript
// validateForSubmit() — used in required-field check (low risk, display-only)
sellingPrice: Number(contract.sellingPrice),
downPayment: Number(contract.downPayment),
monthlyPayment: Number(contract.monthlyPayment),

// detectSignificantChange() — comparison using Number() (higher risk)
Number(dto.interestRate) !== Number(contract.interestRate)
sellingPrice !== Number(contract.sellingPrice)
downPayment !== Number(contract.downPayment)
```

`contract.sellingPrice`, `.downPayment`, `.monthlyPayment` are `Decimal` from Prisma. Converting to `Number()` can lose precision for values > 15 significant digits. The comparison at lines 563–566 risks a false-positive "no change" for large Baht amounts.

**Fix:** 
- For comparisons: use `new Prisma.Decimal(dto.interestRate).equals(contract.interestRate)`
- For display (validateForSubmit): `.toString()` is safer, but `Number()` is acceptable if `checkRequiredContractFields` only checks for existence
