คู่มือสำหรับฝ่ายบัญชี — ครอบคลุมตั้งแต่กรอกฟอร์มซื้อสินทรัพย์ → JE → ค่าเสื่อมรายเดือน → จำหน่าย

เวอร์ชั่น: v3 (deployed 2026-05-11) มาตรฐาน: TAS 16 (PPE) · TFRS for NPAEs · ผังบัญชี CPA Phase A.4 กลุ่มบัญชี: 12-21XX (PPE: odd = cost, even = contra Acc. Depr.) ใช้แทน: journey-asset-module.html

สรุปสำคัญ — สิ่งที่ฝ่ายบัญชีต้องรู้

  • ระบบ ระบุ "ราคาทุน" = basePrice + shippingCost + installationCost + otherCapitalized ตาม TAS 16 ¶16 — ค่าขนส่ง, ค่าติดตั้ง, ค่าทดสอบ ทุกอย่างที่ จำเป็นต่อการใช้งาน → capitalize เข้าทุน (ไม่ลงเป็นค่าใช้จ่าย)
  • VAT 7% — เลือก inclusive/exclusive ได้ ระบบคำนวณ VAT amount ให้เอง Dr 11-4101 (ภาษีซื้อ) แยกบรรทัด ไม่ปนเข้าทุน
  • WHT (หัก ณ ที่จ่าย) — เลือกได้ ภ.ง.ด.3 (บุคคล) หรือ ภ.ง.ด.53 (นิติบุคคล) · หักเฉพาะ "ค่าบริการ" (ม.50 ทวิ, ทป.4/2528) — ไม่หัก WHT จากราคาสินค้า
  • ค่าเสื่อมราคา — Straight-line: (purchaseCost − residualValue) / usefulLifeMonths · อายุใช้งานผู้ใช้กำหนดเอง (ดู พ.ร.ฎ. 145 + TAS 16 ¶57)
  • การจำหน่าย — มี 3 เส้นทาง: ขาย (มีรายได้/VAT), ตัดบัญชี (write-off, ขาดทุน), แลกเปลี่ยน (deferred)
  • กลับรายการ (REVERSE) — เอกสาร POST แล้วถ้าผิด ใช้ Reverse → ระบบสร้าง JE สลับ Dr/Cr ที่ วันที่กลับ ไม่ทับ JE เดิม (TFRS no-touch)
  • เลขเอกสาร auto: ASSET-YYYYMM-NNNN (advisory lock รายเดือน) · เลข JE: JE-YYYYMMDD-NNNNNN
  • ห้าม hard-delete · DRAFT ลบได้, POSTED ทำได้แค่ Reverse · ทุกการเปลี่ยนสถานะมี AuditLog

5ค่าเสื่อมราคารายเดือน

เมื่อ Asset อยู่ในสถานะ POSTED ระบบจะคำนวณค่าเสื่อมราคาให้อัตโนมัติ ทุกสิ้นเดือน (cron: 0 23 28-31 * * Asia/Bangkok) — ผ่าน DepreciationService.runMonthly().

5.1 สูตรคำนวณ — Straight-line (เส้นตรง)

ค่าเสื่อม / เดือน = (purchaseCost − residualValue) / usefulLifeMonths

NBV ปัจจุบัน = purchaseCost − Σ accumulatedDepreciation (≥ residualValue)

ตัวอย่าง: โน้ตบุ๊ก 30,000 ฿ · ซาก 0 · อายุใช้ 36 เดือน → ค่าเสื่อม/เดือน = 30,000 / 36 = 833.33 ฿

  • สร้าง AssetTransferHistory record (เก่า → ใหม่ + reason + วันที่)
  • สร้าง AuditLog action = ASSET_TRANSFERRED

14FAQ — คำถามที่บัญชีถามบ่อย

Q1: ระบบรู้ได้ยังไงว่าจะลง 12-2101 หรือ 12-2103?

จาก field category ที่ผู้บันทึกเลือกใน Section 1:

  • EQUIPMENT → 12-2101 + Acc.Depr. 12-2102 + Dep.Expense 53-1601
  • IMPROVEMENT → 12-2103 + 12-2104 + 53-1602
  • FURNITURE → 12-2105 + 12-2106 + 53-1603
  • VEHICLE → 12-2107 + 12-2108 + 53-1604

Q2: ทำไม Capitalized Cost ใน Live totals ไม่เท่า basePrice?

เพราะ Capitalized Cost = basePrice + shippingCost + installationCost + otherCapitalized (ทุกอย่างที่ capitalizeเข้าทุนตาม TAS 16). ถ้าไม่มีค่าใช้จ่ายอื่น Capitalized Cost = basePrice.

Q3: ถ้าใส่ VAT Inclusive ระบบคำนวณยังไง?

ระบบใช้สูตร: basePriceNet = basePriceInclusive / 1.07 และ vatAmount = basePriceInclusive − basePriceNet. ผลที่ออก: ราคาทุน (Dr 12-21XX) ใช้ basePriceNet, ภาษีซื้อ (Dr 11-4101) = vatAmount.

Q4: WHT หักจากอะไร?

Default = installationCost (ค่าติดตั้ง = ค่าบริการ). ถ้าใบกำกับมี "ค่าบริการ" อื่นด้วย → บัญชี ปรับ whtBaseAmount manual. ห้ามหัก WHT จากราคาสินค้า — ผิดกฎหมาย (ม.50 ทวิ).

Q5: ถ้าซื้อโดยไม่มีใบกำกับภาษี — ลงได้ไหม?

ได้ ตั้ง hasVat = false. ระบบไม่บังคับใบกำกับ แต่ เตือนใน Sticky footer ว่าจะเสีย VAT credit. กรอก "เลขใบกำกับภาษี" เป็นว่างได้.

Q6: ปุ่ม "บันทึก & POST" disable ได้ยังไง?

2 เงื่อนไข AND:

  • errorCount = 0 (V1-V14 ผ่านหมด)
  • Auto Journal balance (Σ Dr = Σ Cr)

Q7: ถ้า POST แล้วลืมกรอกเลขใบกำกับ?

ไม่ใช่ field บังคับ — แก้ในหน้า /assets/:id ได้ตรง field taxInvoiceNo. ไม่กระทบ JE.

Q8: ถ้าซื้อสินทรัพย์มูลค่าน้อย (เช่น สายชาร์จ 200 ฿) — ลง asset หรือ expense?

Policy: ระบบไม่มี threshold บังคับ — ขึ้นกับ accounting judgment. แนะนำ:

  • มูลค่า < 5,000 ฿ และอายุใช้ < 1 ปี → ลง expense module (53-XXXX)
  • มูลค่า ≥ 5,000 ฿ หรือ ใช้ > 1 ปี → ลง asset module (capitalize)

Q9: ค่าเสื่อม cron ทำงานเมื่อไหร่? ถ้าวันสุดท้ายของเดือนตรงกับวันหยุด?

Cron: 0 23 28-31 * * Asia/Bangkok — รัน 4 ครั้ง ทุก 28-31. ระบบมีการเช็คว่า "วันสุดท้ายของเดือน" คือวันไหนจริง (handle ก.พ., เดือน 30 วัน) — JE ที่ post จะมี postedAt ตรงกับ last day of monthเสมอ.

Q10: แก้ไข Asset หลัง POST ได้ field ไหนบ้าง?

ที่หน้า /assets/:id แก้ได้: custodian, location, branchId, serialNo, warrantyExpire, note. แก้ไม่ได้: basePrice, category, residualValue, usefulLifeMonths, VAT/WHT config, payment account — เพราะกระทบ JE. ถ้าต้องแก้ → Reverse แล้วสร้างใหม่.
