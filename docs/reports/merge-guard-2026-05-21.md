### Issues

#### ⚠️ Warning — `Number(input.downAmount)` on money value (`sales-bot.service.ts`)

**File:** `apps/api/src/modules/sales-bot/sales-bot.service.ts`  
**Pattern:**
```typescript
downAmount: Number(input.downAmount ?? 0),
```

`downAmount` represents a cash amount used to generate a PromptPay QR code. While it is **not** stored in a Prisma `Decimal` database column (it goes into an `AuditLog.newValue` JSON blob and into `promptpay-qr`'s `generatePayload()`), using raw `Number()` for a monetary value risks floating-point precision issues in the QR payload (e.g., 100.001 instead of 100.00). The `promptpay-qr` library accepts a plain JS number, so this works today, but a value like `1500.10` from the AI could yield `1500.1000000000001`.

**Suggested fix:**
```typescript
downAmount: Math.round(Number(input.downAmount ?? 0) * 100) / 100,
```
Or use `parseFloat(Number(input.downAmount ?? 0).toFixed(2))` to clamp to 2 decimal places before passing to QR generation.

**Severity: Warning** — no DB precision corruption risk, but could produce incorrect QR amounts for non-round-number down payments.
