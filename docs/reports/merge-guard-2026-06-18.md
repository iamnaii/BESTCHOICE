### 🟡 WARNING — 3 Issues

#### W1 — Missing roles on `GET /accounting/period-status`
**File**: `apps/api/src/modules/accounting/accounting.controller.ts`

```ts
@Get('period-status')
@Roles('OWNER', 'VIEWER')   // ← Missing FINANCE_MANAGER and ACCOUNTANT
getPeriodStatus() {
  return this.service.getAccountingPeriodStatus();
}
```

All other accounting read endpoints (trial-balance, profit-loss) include `'FINANCE_MANAGER', 'ACCOUNTANT'`. `FINANCE_MANAGER` and `ACCOUNTANT` users will get 403 when the UI calls this endpoint. `VIEWER` role is gated behind `viewer_role_enabled` SystemConfig flag (default `false`) so the only role that currently works is `OWNER`.

**Fix**: Change to `@Roles('OWNER', 'FINANCE_MANAGER', 'ACCOUNTANT', 'VIEWER')`.

#### W2 — `Number()` on `Decimal` financial fields in reporting paths
**Files**:
- `apps/api/src/modules/accounting/accounting.service.ts`
- `apps/api/src/modules/contracts/contracts.service.ts`

```ts
// accounting.service.ts — aging/overdue report
const remaining = Number(p.amountDue) - Number(p.amountPaid ?? 0);
// ... then accumulated into bucket sums: summary[bucket] += remaining;

// contracts.service.ts — getMilestonesSummary() dashboard stat
const newThisMonthSum = newContracts.reduce(
  (acc, c) => acc + Number(c.financedAmount),
  0,
);
```

v4 hardening eliminated all `Number(_sum` patterns (53 converted to `Prisma.Decimal`). These are regressions in reporting/dashboard paths — not ledger or payment recording paths, so precision loss risk is low at Thai amounts. However they violate the project rule and the v4 contract.

**Fix**: Use `Prisma.Decimal` accumulation. E.g.:
```ts
import { Prisma } from '@prisma/client';
const remaining = new Prisma.Decimal(p.amountDue ?? 0)
  .minus(new Prisma.Decimal(p.amountPaid ?? 0));
// ...
summary[bucket] = new Prisma.Decimal(summary[bucket]).plus(remaining).toNumber();
```
