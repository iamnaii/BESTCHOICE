# Skill: ตรวจสอบระบบบัญชีตามหลักบัญชีไทย

ตรวจสอบระบบบัญชี BESTCHOICE ทั้งหมดว่าถูกต้องตามมาตรฐานการบัญชีไทยและกฎหมายภาษีอากร

## ขั้นตอน

### 1. อ่าน Prompt ตรวจสอบ
อ่าน `docs/prompts/ACCOUNTING-AUDIT-PROMPT.md` เพื่อดู Checklist ทั้ง 12 หมวดและรูปแบบรายงาน

### 2. ตรวจสอบทีละหมวด (12 หมวด)

ใช้ subagent แบบ parallel เพื่อความเร็ว — แบ่งเป็น 4 กลุ่ม:

**กลุ่ม A: ผังบัญชี + รายได้ + ภาษี**
- หมวด 1: ผังบัญชี (Chart of Accounts)
- หมวด 2: การรับรู้รายได้ (Revenue Recognition)
- หมวด 3: ภาษีมูลค่าเพิ่ม (VAT)
- หมวด 4: ภาษีหัก ณ ที่จ่าย (Withholding Tax)

**กลุ่ม B: ลูกหนี้ + ใบเสร็จ + ต้นทุน**
- หมวด 5: ลูกหนี้เช่าซื้อ / สำรองหนี้สงสัยจะสูญ
- หมวด 6: ใบเสร็จรับเงิน / ใบกำกับภาษี
- หมวด 7: ต้นทุนขาย (COGS)

**กลุ่ม C: ค่าใช้จ่าย + บัญชีระหว่างกัน + รายงาน**
- หมวด 8: ค่าใช้จ่ายและ Workflow อนุมัติ
- หมวด 9: บัญชีระหว่างกัน (Inter-Company)
- หมวด 10: รายงานทางการเงิน

**กลุ่ม D: ชำระเงิน + Audit Trail**
- หมวด 11: การชำระเงินและการจัดสรร
- หมวด 12: Audit Trail และการควบคุมภายใน

สำหรับแต่ละหมวด:
1. อ่านทุกไฟล์ที่ระบุใน Prompt
2. ตรวจตาม Checklist ทุกข้อ
3. บันทึกผล: PASS / FAIL / PARTIAL พร้อมรายละเอียด

### 3. จัดทำรายงานผลตรวจสอบ

สร้างรายงานตามรูปแบบใน Prompt:

```
# รายงานผลตรวจสอบระบบบัญชี BESTCHOICE
วันที่ตรวจสอบ: [วันที่]

## สรุปผลรวม (ตาราง 12 หมวด)

## ประเด็นวิกฤต (Critical) — ต้องแก้ไขทันที
## ประเด็นเตือน (Warning) — ควรแก้ไข
## ข้อเสนอแนะ (Recommendation) — ควรปรับปรุง
## สิ่งที่ทำได้ดี (Good Practices)
## Action Items (ตารางพร้อมความเร่งด่วน)
```

บันทึกรายงานที่ `docs/reports/ACCOUNTING-AUDIT-[วันที่].md`

### 4. สรุปให้ user

แสดงสรุป:
- จำนวน Critical / Warning / Recommendation
- หมวดที่ผ่าน vs ไม่ผ่าน
- Top 3 ประเด็นเร่งด่วนที่ต้องแก้ก่อน

## ขอบเขตที่ไม่ต้องตรวจ
- ดอกเบี้ย flat rate กับ พ.ร.บ.ผ่อนชำระ
- Performance / Scalability
- UI/UX Design
- Security vulnerabilities (มี audit แยก)

## เกณฑ์ PASS / FAIL / PARTIAL

| ผลลัพธ์ | เกณฑ์ |
|---|---|
| **PASS** | ทำตาม checklist ครบ ไม่มี deviation |
| **PARTIAL** | ทำบางข้อ แต่มี gaps ที่ไม่กระทบ legal compliance |
| **FAIL** | ขาด critical item ที่กระทบความถูกต้องทางบัญชี/ภาษี |

## Prerequisite
- ต้องมีไฟล์ `docs/prompts/ACCOUNTING-AUDIT-PROMPT.md` — ถ้าไม่มีให้แจ้ง user ก่อนเริ่ม
