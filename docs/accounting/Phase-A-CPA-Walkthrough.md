## 1. ภาพรวมธุรกิจ

### 1.1 รูปแบบธุรกิจ

BESTCHOICE คือธุรกิจ **ขายมือถือเงินผ่อน** ที่ดำเนินงานในรูปแบบ **1 นิติบุคคล แบ่ง 2 หน่วยธุรกิจ** (จะแยกเป็น 2 นิติบุคคลในอนาคต)

| หน่วย | บทบาท | จด VAT |
|---|---|---|
| **BESTCHOICE SHOP** | ขายมือถือ (ใหม่ + มือสอง) หลายสาขา | ❌ ไม่จด |
| **BESTCHOICE FINANCE** | ปล่อยสินเชื่อเช่าซื้อ ส่วนกลาง | ✅ จด |

### 1.2 ขั้นตอนการเงินเมื่อขายผ่อน

```
ลูกค้า                    SHOP                       FINANCE
  │                        │                           │
  │ ─── 1. เลือกสินค้า ───→│                           │
  │                        │                           │
  │ ─── 2. ดาวน์ ─────────→│ (เก็บเงินดาวน์)           │
  │                        │                           │
  │                        │ ─── 3. โอนสินค้า ───────→ │ (กรรมสิทธิ์ย้าย)
  │                        │                           │
  │                        │←── 4. รับเงิน + คอม ──── │ (FINANCE จ่าย SHOP)
  │                        │                           │
  │ ←── 5. รับเครื่อง ─────│                           │
  │                        │                           │
  │ ─── 6. ผ่อนงวดละ ─────────────────────────────────→│ (LINE LIFF / QR)
  │                        │                           │
  │ (ผ่อนครบ — กรรมสิทธิ์กลับมาเป็นของลูกค้า)        │
```

### 1.3 ตัวอย่างเลข (ใช้ทั้งเอกสารนี้)

มือถือใหม่ราคา **10,000 บาท**:

| รายการ | ยอด (บาท) |
|---|---|
| ราคาขาย | 10,000.00 |
| เงินดาวน์ | 0.00 |
| ดอกเบี้ยรวม 12 เดือน | 1,200.00 |
| ค่าคอมมิชชัน (SHOP รับจาก FINANCE) | 600.00 |
| ภาษีมูลค่าเพิ่ม | 0.00 (ตัวอย่าง — ปกติ 7% บนดอก+คอม) |
| **ยอดผ่อนรวม** | **11,800.00** |
| ผ่อน 12 งวด งวดละ | ~983.33 |
| ต้นทุนสินค้า | 8,000.00 |

## 2. โครงสร้าง 2 บริษัทใน 1 นิติบุคคล

### 2.1 ทำไม 2 ผังบัญชี

แม้จะเป็น 1 นิติบุคคล แต่เก็บบัญชีแยกเพราะ:
1. **VAT แยกชัด** — SHOP ไม่จด VAT, FINANCE จด VAT (กรมสรรพากรต้องเห็นแยก)
2. **เตรียมแยกนิติบุคคลในอนาคต** — เมื่อแยกจริงไม่ต้องรื้อระบบ
3. **Inter-company tracking** — รู้ว่า FINANCE เป็นหนี้ SHOP เท่าไร

### 2.2 ระบบบันทึกบัญชีระหว่างหน่วย

ทุกรายการที่ข้ามฝั่ง SHOP↔FINANCE ใช้ **บัญชีคู่ขนานระหว่างหน่วย (Inter-company Clearing Accounts)**:

| ฝั่ง | รหัสบัญชี | ชื่อ | ประเภท |
|---|---|---|---|
| SHOP | `11-2105` | ลูกหนี้คู่ค้า — FINANCE | สินทรัพย์ |
| FINANCE | `21-1102` | เจ้าหนี้คู่ค้า — SHOP | หนี้สิน |

> 📌 **กฎเหล็ก (Inter-company Invariant)**:
>
> ยอด `11-2105` ฝั่ง SHOP **ต้องเท่ากับ** ยอด `21-1102` ฝั่ง FINANCE เสมอ
>
> ระบบมีหน้า `/accounting/intercompany` แสดงสองยอดพร้อมกัน และเตือนทันทีถ้าไม่ตรง

### 2.3 การชำระเงินระหว่างหน่วย

เมื่อ FINANCE โอนเงินจริงให้ SHOP (เช่น สรุปโอนทุกสิ้นเดือน) → ใช้หน้า **"ชำระเงินระหว่างบริษัท"**:

```
FINANCE: เดบิต 21-1102 เจ้าหนี้คู่ค้า  / เครดิต 11-1101 เงินสด
SHOP:    เดบิต 11-1101 เงินสด        / เครดิต 11-2105 ลูกหนี้คู่ค้า
```

ทั้งสองรายการบันทึกพร้อมกันในธุรกรรมเดียว — ผูกกันด้วยรหัสอ้างอิง `[IC-<uuid>]` ในคำอธิบายรายการ

## 3. นโยบายการบันทึกบัญชี

### 3.1 ตามมาตรฐาน TFRS for NPAEs

| หัวข้อ | นโยบาย |
|---|---|
| **การรับรู้รายได้** | เกณฑ์เงินสดสำหรับรายได้ดอกเบี้ย/ค่าคอม — รับรู้เมื่อลูกค้าจ่ายงวด |
| **การรับรู้ค่าใช้จ่าย** | เกณฑ์คงค้าง — รับรู้เมื่อเกิดรายการ |
| **วิธีคำนวณดอกเบี้ย** | เส้นตรง (Straight-line) ไม่ใช้วิธีอัตราดอกเบี้ยที่แท้จริง |
| **สินค้าคงเหลือ** | เกณฑ์ราคาทุน |
| **หนี้สงสัยจะสูญ** | วิธีตั้งค่าเผื่อ (Allowance Method) ตามช่วงอายุหนี้ |

### 3.2 รายได้รอตัดบัญชี (Deferred Income — ดอกเบี้ย/คอมรอตัดบัญชี)

เพื่อให้ตรงกับเกณฑ์เงินสด แต่บัญชีลูกหนี้เช่าซื้อยังต้องตั้งเต็มยอด:

**ตอนเปิดสัญญา:**
- เดบิต ลูกหนี้เช่าซื้อ **11,800** (ยอดรวมที่ลูกค้าต้องจ่าย)
- เครดิต รายได้ดอกเบี้ยรอตัดบัญชี **1,200** (บัญชี `21-2202`)
- เครดิต รายได้ค่านายหน้ารอตัดบัญชี **600** (บัญชี `21-2201`)
- เครดิต ภาษีขายรอเรียกเก็บ (บัญชี `21-2102`)

**ทุกครั้งที่ลูกค้าผ่อนงวด:**
- เดบิต ดอกเบี้ยรอตัดบัญชี / เครดิต รายได้ดอกเบี้ย (ย้ายจาก "รอ" → "รับรู้แล้ว")
- เดบิต คอมมิชชันรอตัดบัญชี / เครดิต รายได้ค่านายหน้า
- เดบิต ภาษีขายรอเรียกเก็บ / เครดิต ภาษีขาย

> ✅ ระบบทำให้ลูกหนี้เช่าซื้อเป็น **0 บาทพอดี** เมื่อลูกค้าผ่อนครบ
> ✅ รายได้ดอกเบี้ยในงบกำไรขาดทุนแสดง **ตามที่เก็บได้จริง** ไม่ใช่ทั้งก้อนตั้งแต่เปิดสัญญา

### 3.3 นโยบายภาษีมูลค่าเพิ่ม

| รายการ | คิดภาษี? | หมายเหตุ |
|---|---|---|
| ขายเครื่อง (SHOP) | ❌ ไม่คิด | SHOP ไม่จดทะเบียนภาษีมูลค่าเพิ่ม |
| ดอกเบี้ยเช่าซื้อ | ✅ คิด 7% | ⚠️ **รอผู้สอบบัญชียืนยัน** ตาม ม.81(1)(ช) |
| ค่าคอมมิชชัน | ✅ คิด 7% | |
| ค่าปรับล่าช้า | ❌ ไม่คิด | ค่าปรับไม่ใช่ฐานภาษี |

### 3.4 ส่วนลดเมื่อปิดสัญญาก่อนกำหนด

เมื่อลูกค้าปิดสัญญาก่อนกำหนดและได้ส่วนลด:

- **เงินต้น** ต้องชำระเต็ม (ไม่ลดเงินต้น)
- ส่วนลดมาจาก **ดอกเบี้ย + คอมมิชชัน + ภาษี** ตามสัดส่วน
- บันทึกเป็น **ค่าใช้จ่ายชัดเจน** ในบัญชี `53-1805` (FINANCE) และ `53-1801` (SHOP)
- แสดงในงบกำไรขาดทุนเป็น "ส่วนลดให้ลูกค้า" ⚠️ **รอผู้สอบบัญชียืนยัน**: ต้องออกใบลดหนี้หรือไม่?

## 5. ตัวอย่างรายการบัญชี (Journal Entry Templates)

### 5.1 เปิดสัญญา

ลูกค้าซื้อเครื่อง 10,000 ดาวน์ 0 ผ่อน 12 เดือน รวม 11,800 บาท ต้นทุนเครื่อง 8,000

#### ฝั่ง SHOP

| เดบิต/เครดิต | บัญชี | จำนวน |
|---|---|---|
| เดบิต | `11-2105` ลูกหนี้คู่ค้า — FINANCE | 10,600.00 |
| เดบิต | `51-1101` ต้นทุนขาย — ใหม่ | 8,000.00 |
| เครดิต | `41-1101` รายได้ — มือถือใหม่ | 10,000.00 |
| เครดิต | `21-2201` รายได้ค่านายหน้ารอตัดบัญชี | 600.00 |
| เครดิต | `11-3101` สินค้าคงเหลือ — ใหม่ | 8,000.00 |

> 💡 **ผลกระทบ**: SHOP ขายเครื่อง รับรู้รายได้ 10,000 ทันที (กรรมสิทธิ์โอน) แต่ค่าคอม 600 ยังไม่รับรู้เป็นรายได้ (รอตัดเป็นรายงวด)
> SHOP มีลูกหนี้คู่ค้า 10,600 บาทที่ FINANCE ต้องจ่ายให้ (= ราคาขาย + ค่าคอม − ดาวน์)

#### ฝั่ง FINANCE

| เดบิต/เครดิต | บัญชี | จำนวน |
|---|---|---|
| เดบิต | `11-2102` ลูกหนี้เช่าซื้อ | 11,800.00 |
| เครดิต | `21-1102` เจ้าหนี้คู่ค้า — SHOP | 10,600.00 |
| เครดิต | `21-2202` รายได้ดอกเบี้ยรอตัดบัญชี | 1,200.00 |

## 8. คำถามที่ขอให้ผู้สอบบัญชีช่วยตอบ

ก่อนเชื่อมต่อโปรแกรม PEAK และยื่นภาษี ขอให้ผู้สอบบัญชีตอบคำถามต่อไปนี้:

### ❓ คำถามที่ 1 — ภาษีมูลค่าเพิ่มบนดอกเบี้ยเช่าซื้อ

ตาม **มาตรา 81(1)(ช) แห่งประมวลรัษฎากร** ดอกเบี้ยจากการให้กู้ยืมเงินได้รับยกเว้นภาษีมูลค่าเพิ่ม

**คำถาม**: ดอกเบี้ยเช่าซื้อถือเป็น "ดอกเบี้ยจากการกู้ยืม" หรือ "ค่าตอบแทนการเช่าซื้อ"?

ถ้า **ได้รับยกเว้นภาษี** → ต้องเอาภาษีออกจากรายการบัญชี (ลบ `21-2102` และ `21-2101` ลงตาม)
ถ้า **คิดภาษี 7%** → ระบบปัจจุบันถูกต้อง

### ❓ คำถามที่ 2 — ใบกำกับภาษีออกตอนไหน

**คำถาม**: ออกใบกำกับภาษีตอนไหน?
- **(ก)** เปิดสัญญาทั้งก้อน (1 ใบครอบคลุมทุกงวด) → ต้องออกใบลดหนี้เมื่อมีส่วนลดล่วงหน้า
- **(ข)** รายงวด (12 ใบ ออกตอนรับเงิน) → ระบบปัจจุบันถูกต้อง

### ❓ คำถามที่ 3 — ส่วนลดให้ลูกค้าต้องออกใบลดหนี้หรือไม่?

เมื่อปิดสัญญาก่อนกำหนดและให้ส่วนลด ระบบบันทึก `53-1805 ส่วนลดให้ลูกค้า`

**คำถาม**:
- ถ้าตอบคำถามที่ 2 = (ก) → ต้องออกใบลดหนี้ส่งกรมสรรพากรหรือไม่?
- ถ้าตอบคำถามที่ 2 = (ข) → ส่วนลดเกิดก่อนการออกใบกำกับ → ไม่ต้องออกใบลดหนี้ ใช่หรือไม่?

### ❓ คำถามที่ 4 — รับรู้รายได้ขายตอนไหน

ระบบปัจจุบัน: SHOP รับรู้ `41-1101 รายได้มือถือใหม่` **ตอนเปิดสัญญา** (กรรมสิทธิ์โอนจาก SHOP → FINANCE)

**คำถาม**: ถูกต้องตาม TFRS for NPAEs หรือควรรับรู้เมื่อ:
- ลูกค้าผ่อนครบ
- กรรมสิทธิ์ตกเป็นของลูกค้า

### ❓ คำถามที่ 5 — บัญชีระหว่างหน่วยใน 1 นิติบุคคล

ระบบใช้ 2 ผังบัญชี (SHOP + FINANCE) ในนิติบุคคลเดียวกัน

**คำถาม**:
- ทำได้ตามมาตรฐาน TFRS for NPAEs หรือไม่?
- ต้องรวมงบเป็น 1 ฉบับ (consolidate) ตอนปิดงบประจำปีหรือไม่?
- ถ้ารวมงบ บัญชีคู่ขนานระหว่างหน่วย (เจ้าหนี้/ลูกหนี้คู่ค้า) จะหักกลบกันเอง — ต้องเปิดเผยแยกหรือไม่?

### ❓ คำถามที่ 6 — การจัดประเภทรายได้รอตัดบัญชี

ระบบใช้ `21-2202 รายได้ดอกเบี้ยรอตัดบัญชี` และ `21-2201 รายได้ค่านายหน้ารอตัดบัญชี` เป็นหนี้สิน

**คำถาม**:
- จัดประเภทเป็น **หนี้สินหมุนเวียน** หรือ **หนี้สินไม่หมุนเวียน** ตามอายุสัญญา?
- รายงานในงบดุลใช้ชื่อ "รายได้รับล่วงหน้า" หรือ "รายได้รอตัดบัญชี"?

### ❓ คำถามที่ 7 — อัตราการตั้งค่าเผื่อหนี้สงสัยจะสูญ

อัตราค่าเผื่อตามช่วงอายุหนี้:

| 31–60 วัน | 5% |

| 91–180 วัน | 50% |

**คำถาม**: เหมาะสมตามมาตรฐาน TFRS for NPAEs และนโยบายของกรมสรรพากรหรือไม่?

### ❓ คำถามที่ 8 — รหัสบัญชีตรงกับ PEAK หรือไม่

ระบบมี mapping `peakAccountCode` (ตรงกับรหัสในผังบัญชีของเรา เช่น `11-1101`)

**คำถาม**: ผังบัญชีมาตรฐานของ PEAK ใช้รหัสแบบนี้หรือเปล่า? ต้องเปลี่ยนรหัสให้ตรงหรือไม่?

## 10. อภิธานศัพท์

| คำ | ความหมาย |
|---|---|
| **ลูกหนี้เช่าซื้อ** | ยอดรวมที่ลูกค้าต้องจ่ายตลอดสัญญา (เงินต้น + ดอกเบี้ย + คอมมิชชัน + ภาษี) |
| **รายได้รอตัดบัญชี** | รายได้ที่ตั้งเป็นหนี้สินก่อน แล้วค่อยย้ายเป็นรายได้ตามงวดที่ลูกค้าจ่าย (ตามเกณฑ์เงินสด) |
| **ภาษีขายรอเรียกเก็บ** | ภาษีมูลค่าเพิ่มที่ยังไม่ได้ออกใบกำกับ — รอจนรับชำระแต่ละงวด |
| **บัญชีคู่ขนานระหว่างหน่วย** | บัญชีคู่ขนาน SHOP↔FINANCE (ลูกหนี้คู่ค้า/เจ้าหนี้คู่ค้า) สำหรับติดตามยอดระหว่างหน่วย |
| **เกณฑ์เงินสด** | รับรู้รายได้/ค่าใช้จ่ายเมื่อมีเงินสดเข้า/ออก (ใช้กับรายได้ดอกเบี้ย/ค่าคอม) |
| **เกณฑ์คงค้าง** | รับรู้เมื่อเกิดรายการ ไม่ขึ้นกับเงินสด (ใช้กับค่าใช้จ่าย) |
| **เฟส A** | ชุดการแก้ไขระบบบัญชีระหว่าง 29 เม.ย. – 30 เม.ย. 2569 (Pull Request #722–#728) |
| **TFRS for NPAEs** | มาตรฐานรายงานทางการเงินสำหรับกิจการที่ไม่มีส่วนได้เสียสาธารณะ |
| **รายการบัญชี (Journal Entry)** | รายการบันทึกบัญชี ประกอบด้วยเดบิตและเครดิตที่สมดุลกัน |
| **ผังบัญชี (Chart of Accounts)** | บัญชีรหัสและชื่อบัญชีทั้งหมดที่ใช้ในระบบบัญชี |
| **PEAK** | โปรแกรมบัญชีออนไลน์ของไทย (peakaccount.com) ที่ระบบจะส่งรายการบัญชีไป sync |

## ภาคผนวก ข — ระบบการบันทึกและการตรวจสอบ

### ระบบบันทึกรายการบัญชีอัตโนมัติเมื่อเกิดเหตุการณ์ต่อไปนี้:

| เหตุการณ์ | ประเภทรายการบัญชี |
|---|---|
| เปิดสัญญา | CONTRACT |
| รับชำระค่างวด | PAYMENT |
| รับเงินเกิน | CUSTOMER_CREDIT_OVERPAY |
| ใช้เครดิตลูกค้าตัดงวด | CREDIT_ALLOCATION |
| ปิดสัญญาก่อนกำหนด | EARLY_PAYOFF |
| ตัดหนี้สูญ | BAD_DEBT_WRITEOFF |
| ตั้งค่าเผื่อหนี้สงสัยจะสูญ | BAD_DEBT_PROVISION |
| ขายสินค้ายึดคืน | REPO_RESALE |
| ชำระเงินระหว่างหน่วย | IC_SETTLEMENT |
| จ่ายค่าใช้จ่าย | EXPENSE |

### ระบบตรวจสอบ:
- ทุกรายการบัญชีมีรหัสอ้างอิงเชื่อมไปยังเอกสารต้นเรื่อง (เช่น เลขสัญญา, เลขใบเสร็จ)
- รายการที่บันทึกแล้วเป็นถาวร แก้ไขไม่ได้
- มีบันทึกการตรวจสอบแยกสำหรับ การโพสต์/ยกเลิก/เปิดบัญชีใหม่
- ทุกรายการตรวจสมดุล (เดบิต = เครดิต ภายใน 0.01 บาท) ก่อนบันทึก พร้อมระบบเตือนถ้าผิด
