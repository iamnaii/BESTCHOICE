### Issues Found

#### Warning

**W1 — No timeout/AbortController on Vertex AI fetch**  
File: `apps/api/src/modules/staff-chat/services/embedding.service.ts`  
The `embedBatch()` method calls the Vertex AI embedding endpoint using raw `fetch()` with no timeout or `AbortController`. A slow or hanging Vertex AI response will block the AI suggestion code path indefinitely. v3 hardening (PR #437) established the pattern: 15s `AbortSignal.timeout()` + Sentry capture on abort. The same pattern should be applied here.

```typescript
// Current (risky):
const res = await fetch(this.endpoint, { method: 'POST', headers: {...}, body: ... });

// Should be:
const res = await fetch(this.endpoint, {
  method: 'POST',
  headers: {...},
  body: ...,
  signal: AbortSignal.timeout(15_000),
});
// + Sentry.captureException on timeout
```
