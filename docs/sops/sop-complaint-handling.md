# SOP: Complaint Handling
**เวอร์ชัน:** 1.0
**เจ้าของ:** COO
**อัปเดตล่าสุด:** 2026-04-01
**SLA:** แก้ไขภายใน **24 ชั่วโมง** (Level 1-2), **4 ชั่วโมง** (Level 3)

---

## 1. ระดับ Complaint

### Level 1 — ร้องเรียนทั่วไป
- ความล่าช้าในการตอบกลับ
- ข้อมูลผิดพลาดเล็กน้อย
- สอบถามขอรายละเอียดเพิ่มเติม

**ผู้รับผิดชอบ:** Customer Service (น้องเบส) / SALES
**SLA:** 8 ชั่วโมง

### Level 2 — ร้องเรียนระดับปานกลาง
- ไม่พอใจการบริการ
- ข้อมูลสัญญาผิดพลาด
- การชำระเงินมีปัญหา (บันทึกผิด)
- ขอยกเว้นค่าปรับครั้งแรก

**ผู้รับผิดชอบ:** BRANCH_MANAGER
**SLA:** 24 ชั่วโมง

### Level 3 — ร้องเรียนรุนแรง / เร่งด่วน
- ร้องเรียนซ้ำ (แก้ไขแล้วแต่ปัญหายังอยู่)
- ข้อพิพาทยอดเงิน
- ภัยคุกคามทางกฎหมาย
- ร้องเรียนต่อหน่วยงานภายนอก (สคบ., ธปท.)
- สื่อมวลชน

**ผู้รับผิดชอบ:** COO
**SLA:** 4 ชั่วโมง

---

## 2. กระบวนการรับ Complaint

### 2.1 รับเรื่อง (T=0)
1. รับทราบและยืนยันกับลูกค้าทันที
2. บันทึกข้อมูลใน system (หรือแบบฟอร์ม complaint log)
3. แจ้งลูกค้า: ชื่อผู้รับผิดชอบ + timeline ที่คาดว่าจะแก้ไขได้

**ตัวอย่าง acknowledgment:**
```
เรียนคุณ [ชื่อ]
ขอบคุณที่แจ้งปัญหามานะครับ/ค่ะ เราได้รับเรื่องของคุณแล้ว
รหัสการร้องเรียน: [REF-XXXXX]
ผู้ดูแล: [ชื่อ]
คาดว่าจะได้รับการติดต่อกลับภายใน [X ชั่วโมง]
หากมีข้อสงสัย กรุณาแจ้ง REF ข้างต้นครับ/ค่ะ
```

### 2.2 สืบสวน (T+1 ถึง T+SLA)
1. รวบรวมข้อมูลที่เกี่ยวข้อง (สัญญา, history การชำระ, บันทึกการสนทนา)
2. ระบุ root cause (ดู Section 3)
3. เสนอทางแก้ไข

### 2.3 แก้ไข
1. ดำเนินการแก้ไขตามทางเลือกที่เหมาะสม
2. แจ้งลูกค้า + รอ confirmation
3. บันทึกผลใน complaint log

### 2.4 ปิดเรื่อง
- ได้รับ confirmation จากลูกค้าว่าพอใจ
- หรือผ่าน SLA แล้ว (ต้องบันทึกเหตุผล)
- ส่ง satisfaction survey

## 3. Root Cause Analysis Template

```
## Complaint Analysis
วันที่รับเรื่อง: [วันที่]
ระดับ: Level [1/2/3]
รหัส: REF-XXXXX

### ปัญหาที่รายงาน
[สรุปปัญหาที่ลูกค้าแจ้ง]

### Root Cause
- [ ] Human error (พนักงาน)
- [ ] Process gap (กระบวนการไม่ชัดเจน)
- [ ] System bug (ระบบมีปัญหา)
- [ ] External factor (ปัจจัยภายนอก)
- [ ] Other: [ระบุ]

### การแก้ไขครั้งนี้
[อธิบายสิ่งที่ทำ]

### การป้องกันซ้ำ
[มาตรการป้องกัน / อัปเดต SOP / แจ้ง CTO]
```

## 4. Escalation Matrix

```
Customer Service (น้องเบส)
    ↓ (ถ้าแก้ไม่ได้ใน 3 ชม.)
BRANCH_MANAGER
    ↓ (ถ้าแก้ไม่ได้ใน 20 ชม. หรือ Level 3)
COO
    ↓ (ถ้าเป็นคดีความหรือผลกระทบใหญ่)
Board/Legal
```

## 5. Complaint Tracking

| Field | รายละเอียด |
|-------|-----------|
| REF | REF-YYYYMMDD-XXX |
| วันที่รับ | วันที่รับ complaint |
| ช่องทาง | LINE / โทรศัพท์ / สาขา |
| ประเภท | ระดับ 1/2/3 |
| สถานะ | รับเรื่อง / กำลังดำเนินการ / แก้ไขแล้ว / ปิด |

| วันที่ปิด | วันที่ปิดเรื่อง |
| Root Cause | [ประเภท] |

## 6. การวัดผล

- **เป้าหมาย:** ≥ 90% ของ complaints แก้ไขได้ภายใน SLA
- **First Contact Resolution Rate:** ≥ 70%
- **รายงาน:** รายสัปดาห์ ส่ง COO
