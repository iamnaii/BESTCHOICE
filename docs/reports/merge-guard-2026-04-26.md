#### ⚠️ Warning — No Database Transaction Around Create Loop

**File:** `scripts/seed-collections-test-data.ts` lines 176–246  
**Detail:** Each iteration of the `for` loop creates a customer, then a contract, then payments as three separate Prisma calls outside any `$transaction`. If the process crashes after creating a customer but before its contract (or after the contract but before `payment.createMany`), orphaned records are left with the MARKER but no associated contract/payments.

The cleanup script handles orphaned customers (it checks `contracts.length === 0` → eligible for cleanup), but orphaned contracts would not be caught by the cleanup because it only finds contracts with `notes === MARKER` — which are correctly tagged. The risk is low (dev/test usage only), but wrapping each iteration in `await prisma.$transaction([...])` would eliminate it entirely.

**Recommendation:** Wrap each customer+contract+payments create group in a `prisma.$transaction()`.
