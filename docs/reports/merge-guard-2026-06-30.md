**File:** `apps/api/src/modules/purchase-orders/services/po-receiving.service.ts:86`

```ts

costPrice: Number(poItem.unitPrice),
```

`PurchaseOrderItem.unitPrice` is `Decimal @db.Decimal(12, 2)` in Prisma schema. `Product.costPrice` is also `Decimal @db.Decimal(12, 2)`. Converting via `Number()` before writing to the DB violates the project's money handling rule (must preserve `Prisma.Decimal` precision throughout). An identical pattern exists at line 250 (the legacy `goodsReceiving()` path — already present in `main` at line 179), but the branch propagated it into the newly added `receive()` method at line 86.

**Fix:** Use `new Prisma.Decimal(poItem.unitPrice.toString())` or pass the Decimal directly (Prisma accepts Decimal inputs on Decimal columns).
