### Critical Issues

#### C-001: `Number()` on Decimal money fields — **67 occurrences** (BLOCKER)

The project rule is: **use `Prisma.Decimal` for all money arithmetic; never `Number()`**.  
This branch introduces 67 new `Number()` calls on financial data from Prisma `_sum` aggregates and Decimal fields.

Affected files and examples:

**`apps/api/src/modules/accounting/accounting.service.ts`**
```ts
// Line ~getBalanceSheet()
const grossReceivables = Number(hpReceivables._sum.amountDue || 0);
const paidOnReceivables = Number(hpReceivables._sum.amountPaid || 0);
const allowanceForDoubtful = Number(provisions._sum.provisionAmount || 0);
const inventoryValue = Number(inventory._sum.costPrice || 0);
// ... 20+ more in getCashFlowStatement()
const cashFromInstallments = Number(installmentPayments._sum.amountPaid || 0);
```

**`apps/api/src/modules/reports/reports.service.ts`**
```ts
const shopProfit = Number(t.shopProfit);
const financeProfit = Number(t.financeProfit);
const commission = Number(t.commission);
```

**`apps/api/src/modules/accounting/bad-debt.service.ts`**
```ts
const remaining = Number(p.amountDue) - Number(p.amountPaid);
summary.totalOutstanding += Number(p.outstandingAmount);
```

**`apps/api/src/modules/sales/sales.service.ts`**
```ts
const costPrice = product ? Number(product.costPrice) : 0;
```

**Fix required:** Replace `Number(x._sum.field || 0)` with `new Prisma.Decimal(x._sum.field ?? 0)` and use `.add()`, `.sub()`, `.mul()` for arithmetic. For report output objects (plain JSON response), conversion to `number` is acceptable only at the serialization boundary — not mid-calculation.

#### W-002: File upload endpoint in `staff-chat.controller.ts` — MIME type regex allows broad `application/vnd.openxmlformats*` pattern

The `FileTypeValidator` regex is:
```ts
/^(image\/(jpeg|png|webp)|application\/pdf|application\/(msword|vnd\.openxmlformats))/
```
The suffix `vnd.openxmlformats` matches the MIME type prefix but not the full MIME like `vnd.openxmlformats-officedocument.wordprocessingml.document`. The regex may accidentally pass other `vnd.openxmlformats*` MIME types. Recommend tightening to the full MIME strings or using an allowlist array.
