ผลต่อ “ค่าปรับเกินดิว” ล้างโดยปริยาย

หัวใจของกรณีเกินดิว — การเลื่อนวันครบกำหนดทำให้ค่าปรับที่ค้างหายไปเอง

เพราะเลื่อน Payment.dueDate ไปข้างหน้า ตอนลูกค้าจ่ายจริง:

daysOverdue = วันจ่าย − dueDateใหม่ ≤ 0  ⟹  resolveLateFee = 0 ฿

👉 พูดง่ายๆ: ปรับงวด = ยกค่าปรับที่ค้างให้ แล้วเก็บค่าธรรมเนียมเลื่อนแทน — และไม่มี late fee ปลอมเข้า 42-1103

◆ จุดที่ควรรู้เชิงบัญชี

ค่าธรรมเนียมเลื่อนงวด ไม่ได้ลงเป็นรายได้ — มันเข้า 21-1103 (เงินรับล่วงหน้า) แล้วงวดสุดท้ายก็ถูกลดยอดเท่ากันเป๊ะ สุทธิจึงเป็น “การจ่ายบางส่วนของงวดสุดท้ายล่วงหน้า” ไม่ใช่ค่าปรับ/รายได้เพิ่ม และ ไม่คิด VAT

▲ ข้อควรยืนยันกับนักบัญชี

ถ้าเจตนาจริงคืออยากให้ “ค่าธรรมเนียมเลื่อนงวด” เป็น รายได้ (เหมือนค่าบริการ) — ปัจจุบันโค้ด ไม่ได้ ลงเป็นรายได้ มันเป็นแค่การเก็บเงินงวดสุดท้ายมาล่วงหน้า (net-zero) หากต้องการให้เป็นรายได้จริง ต้องปรับเทมเพลต JP6 ให้ Cr บัญชีรายได้แทน 21-1103 และไม่ลด amountDue งวดสุดท้าย — เป็นการตัดสินใจเชิงนโยบาย
