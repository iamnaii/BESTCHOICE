Flow งานประจำวันของแต่ละ Role

วิเคราะห์จาก business model — ช่วยตรวจว่าตรงกับความเป็นจริงไหม

🟢 SALES — พนักงานขาย

Flow หลัก: ขาย → สร้างสัญญา → รับเงิน
1. เช็คสต็อก → 2. ลูกค้าเข้าร้าน → 3. ตรวจเครดิต → 4. ขาย (POS) → 5. สร้างสัญญาผ่อน → 6. รับเงินดาวน์
รับซื้อมือสอง (Trade-In) ดูยอดคอมฯ แชทลูกค้า (Inbox)

หน้าที่ใช้: สต็อก, ลูกค้า, ตรวจเครดิต, POS, สัญญา, ชำระ, Trade-In, คอมมิชชัน, Inbox

🔵 BRANCH_MANAGER — ผจก.สาขา

Flow หลัก: ดูภาพรวม → จัดการสต็อก → ดูแลทีม
1. Dashboard → 2. เช็คสต็อก/แจ้งเตือน → 3. สั่งซื้อ/โอนสต็อก → 4. อนุมัติสัญญา → 5. ดูยอดขาย
ดูรายงาน จัดการ Supplier CRM / Inbox ค้างชำระ

หน้าที่ใช้: Dashboard, สต็อก(+alerts/transfers), PO, สัญญา, ยอดขาย, รายงาน, Supplier, CRM, ค้างชำระ

🟣 FINANCE_MANAGER — ผจก.การเงิน

Flow หลัก: ตรวจค้างชำระ → ติดตาม → รับเงิน → จัดการพอร์ต
1. Dashboard → 2. ลูกค้าค้างชำระ → 3. Collection Dashboard → 4. รับชำระค่างวด → 5. ออกใบเสร็จ
Finance Portfolio Finance Receivable เปลี่ยน/ยึดเครื่อง คอมมิชชัน

หน้าที่ใช้: Dashboard, ค้างชำระ, Collection, ชำระ, ใบเสร็จ, Portfolio, Receivable, Exchange, Repo, คอมฯ

🟡 ACCOUNTANT — ฝ่ายบัญชี

Flow หลัก: บันทึกรายรับ-รายจ่าย → ตรวจสอบ → รายงาน
1. ตรวจชำระ/ใบเสร็จ → 2. บันทึกรายจ่าย → 3. กำไร-ขาดทุน → 4. ภาษี
ปิดบัญชีรายเดือน ผังบัญชี PEAK Sync Financial Audit

หน้าที่ใช้: ชำระ, ใบเสร็จ, รายจ่าย, กำไร-ขาดทุน, ภาษี, Monthly Close, ผังบัญชี, PEAK, Audit

🔴 OWNER — เจ้าของ

Flow หลัก: ภาพรวมธุรกิจ → ตัดสินใจ → ตั้งค่า
1. Dashboard KPIs → 2. รายงาน/กำไร-ขาดทุน → 3. Collection Overview → 4. ตั้งค่า/จัดการระบบ
เข้าถึงทุกหน้า (54 หน้า) ตั้งค่าทั้งหมด (13 เมนู) Ads & ROI

ใช้ทุกหน้า แต่โฟกัส: Dashboard, Reports, P&L, Collection, Settings, Ads

💡 Key Insight

ปัญหาหลัก: เมนูจัดตาม "ประเภทข้อมูล" (สต็อก, สัญญา, ชำระ, ฯลฯ) แต่พนักงานคิดตาม "flow งาน" (ขาย→สร้างสัญญา→รับเงิน)

ทำให้ต้อง click ข้าม section ไปมาตลอด — เช่น SALES ต้องกระโดดระหว่าง "ขาย" → "สัญญา" → "การเงิน" แค่จะปิดการขาย 1 รายการ
