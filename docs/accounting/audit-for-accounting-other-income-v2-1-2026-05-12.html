Validation Rules V1-V14 (เทียบ PDF Spec)

รหัส คำอธิบาย สถานะในระบบ
V1 Dr = Cr (สมุดรายวันสมดุล) ระบบสร้างให้อัตโนมัติ (สมดุลโดยโครงสร้าง)
V2 Journal lines ≥ 2 บรรทัด ระบบสร้างให้อัตโนมัติ
V3 วันที่ออกเอกสาร + ช่องทางชำระ + อย่างน้อย 1 รายการ ครบถ้วน บล็อกการบันทึก
V4 ทุก item ใช้บัญชีกลุ่ม 42-XXXX + จำนวน > 0 บล็อกการบันทึก
V5 Dr XOR Cr (ต่อบรรทัด) ระบบสร้างให้อัตโนมัติ
V6 VAT consistent — มี VAT% > 0 → ยอด VAT ต้อง > 0 บล็อกการบันทึก
V7 WHT% อยู่ในชุด {0, 1, 2, 3, 5, 7, 10, 15} เตือน (ไม่บล็อก) ถ้าใช้ % แปลก
V8 วันที่ออกเอกสารต้องอยู่ในงวดที่ยังเปิด บล็อก POST + Reverse
V9 Maker ≠ Approver (ผู้สร้าง ≠ ผู้อนุมัติ) บังคับใน approve() เมื่อเปิด flag
V10 ผลต่าง > 0 บาท ต้องระบุบัญชีปรับ บล็อกการบันทึก
V11 ยอด ≥ เกณฑ์ที่ตั้งไว้ ต้องแนบไฟล์ประกอบ บล็อกการบันทึก
V12 ผลรวมของบัญชีปรับ = |ผลต่าง| บล็อกการบันทึก
V13 ทุกแถวของบัญชีปรับต้องเลือกบัญชี บล็อกการบันทึก
V14 ทุกแถวของบัญชีปรับ จำนวน > 0 บล็อกการบันทึก
V15 ดอกเบี้ยฝาก 42-1102 ได้รับยกเว้น VAT (ม.81(1)(ฏ)) — VAT% ต้อง = 0 บล็อก + ส่วนขยาย BESTCHOICE
ทำไม V1/V2/V5 ไม่ validate ที่ขั้นบันทึก?
Spec ของฝ่ายบัญชีระบุ V1 (Dr=Cr), V2 (≥ 2 บรรทัด), V5 (Dr XOR Cr) เป็นกฎสำหรับ สมุดรายวัน. ระบบ Other Income ไม่ให้ user พิมพ์สมุดรายวันโดยตรง — ผู้ใช้กรอกแค่ "รายการรายได้ + ยอดที่รับจริง" แล้วระบบสร้างสมุดรายวันให้อัตโนมัติผ่าน AutoJournalService ซึ่ง:
  • สร้างบรรทัด Dr/Cr ครบทุกประเภท (เงินสด, รายได้, VAT, WHT, ปรับ)
  • คำนวณ Dr = Cr โดยสูตร (สมดุลโดยโครงสร้าง — ผิดไม่ได้)
  • ออกบรรทัดทุกบรรทัดเป็น Dr หรือ Cr เฉพาะอย่างใดอย่างหนึ่ง
ดังนั้น V1/V2/V5 ไม่มีโอกาส fail ได้ — ไม่ต้องตรวจซ้ำที่ validation step

นโยบาย WHT 42-1102 (ดอกเบี้ยฝาก)

PDF Gap Analysis หน้า 4 ระบุนโยบาย WHT ตามประเภทเงินฝาก. ระบบปรับปรุงตามคำแนะนำของฝ่ายบัญชี:

ประเภทเงินฝาก ผู้รับ WHT % อ้างอิงกฎหมาย
ออมทรัพย์ นิติบุคคล (BESTCHOICE) 1% ท.ป.4/2528 ข้อ 1
ฝากประจำ ทุกประเภท 15% ม.50
ออมทรัพย์ บุคคลธรรมดา (ดอกเบี้ย > 20,000 บาท/ปี) 15% ม.50

การเปลี่ยนแปลงในระบบ

  • A Default WHT% เปลี่ยนจาก 15% → 1%ตรงเคสจริงของ BESTCHOICE (ออมทรัพย์ KBank, นิติบุคคล)
  • B ลบ warning "WHT ต้องเป็น 15%" ออกเดิมระบบเตือนทุกครั้งที่ไม่ใช่ 15% — ผิดเคสนิติบุคคล. ตอนนี้เลือกได้อย่างอิสระ ตามดุลพินิจของฝ่ายบัญชี
  • C Dropdown WHT% ครบ 8 ค่า0 / 1 / 2 / 3 / 5 / 7 / 10 / 15
  • D คำแนะนำต่อบัญชี (tooltip)เลือก 42-1102 → แสดง "แนะนำ 1% — นิติบุคคล ออมทรัพย์ (ท.ป.4/2528)". เลือก 42-1105 → แสดง "แนะนำ 1% — ขายสินทรัพย์ให้นิติบุคคล". เป็นคำแนะนำเฉย ๆ ไม่บล็อก
  • E กฎ V15 — บล็อก VAT% > 0 สำหรับ 42-1102บังคับให้ VAT% = 0 เสมอ (ม.81(1)(ฏ) — ดอกเบี้ยฝากยกเว้น VAT)
หมายเหตุของ ก ม.
PDF Gap Analysis เสนอ "Default 15% safer" — แต่หลังตรวจสอบเคส BESTCHOICE จริง (KBank ออมทรัพย์ นิติบุคคล) เป็น 1% มากกว่า ทีม Dev จึงเปลี่ยน default เป็น 1% ตาม. กรณีฝากประจำ (15%) ผู้ใช้เลือกจาก dropdown ได้ทุกเมื่อ
