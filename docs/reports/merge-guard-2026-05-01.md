#### Warning — `Number()` on Decimal field in repossessions.service.ts

**File**: `apps/api/src/modules/repossessions/repossessions.service.ts` (line ~398)

```typescript
const resellPrice = new Prisma.Decimal(
  dto.resellPrice ?? Number(repo.resellPrice ?? 0),  // ← WARNING
);
```

`repo.resellPrice` is a `Prisma.Decimal` from the database. Wrapping it in `Number()` converts it to an IEEE 754 float before passing to `Prisma.Decimal()`, which can introduce precision loss on large values. Should use:

```typescript
const resellPrice = new Prisma.Decimal(dto.resellPrice ?? repo.resellPrice ?? 0);
```

`Prisma.Decimal` constructor accepts `Prisma.Decimal` directly.

This is acceptable because the JE lines store JS `number` for the Prisma `Float` (or `Decimal`) schema field, and all arithmetic prior to conversion uses `Prisma.Decimal`. No precision loss risk for amounts ≤ 12 digits with 2 decimal places (within IEEE 754 safe range for baht).
