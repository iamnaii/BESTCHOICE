8 จุดที่ต้องระวัง

1

2A เป็น idempotent ผ่าน accrualJournalEntryId

รัน cron ซ้ำได้ปลอดภัย — ไม่สร้าง JE ซ้ำ

2

Tolerance ≤ 1 บาท

Overpay → auto โพสต์ 53-1503 · Underpay → ต้อง approver (OWNER / ACCOUNTANT / BRANCH_MANAGER)

3

Cash dimension required

ทุก Payment ต้องเลือก 1 ใน 6 บัญชี: 11-1101/02/03 (เงินสดต่อคน) หรือ 11-1201/02/03 (ธนาคาร)

4

Late fee VAT exempt

auto post 42-1103 จาก 2B (block ใน Other Income module เพื่อกัน double entry)

5

Unbalanced JE → throw + Sentry

PR #444 fix — เคยเป็น silent return null ทำให้ silent loss · ตอนนี้ throw ทันทีถ้า Dr ≠ Cr
