## Summary

Branch adds inter-company transaction tracking (SHOP↔FINANCE) and a balance-sheet / cash-flow report suite. The architecture and guard setup are correct. However, the implementation contains a pervasive precision violation: **every financial aggregation uses `Number()` instead of `Prisma.Decimal`**, undermining the purpose of the accounting audit the branch purports to fix.

---

## Critical Issues (must fix before merge)

### C-001 — `Number()` on Prisma Decimal aggregates (38+ instances, 4 files)

**Severity**: CRITICAL  
**Rule**: `.claude/rules/database.md` — "ใช้ Decimal เท่านั้น … ห้ามใช้ Float หรือ Int สำหรับจำนวนเงิน"  

IEEE 754 floating-point errors will silently accumulate in every balance sheet and cash-flow calculation. With amounts in the ฿100,000–฿1,000,000 range, rounding drift of ±0.01–0.10 per aggregation step is expected.

**Files (new code additions only)**:

`apps/api/src/modules/accounting/accounting.service.ts` — 23 instances:
```ts
bundleCost = bundleProducts.reduce((sum, p) => sum + Number(p.costPrice || 0), 0);
const grossReceivables = Number(hpReceivables._sum.amountDue || 0);
const cashFromInstallments = Number(installmentPayments._sum.amountPaid || 0);
// ... 20 more
```

`apps/api/src/modules/reports/reports.service.ts` — 10 instances:
```ts
const totalLateFees = Number(lateFeeAgg._sum.lateFee || 0);
const shopProfit = Number(t.shopProfit);
const financeProfit = Number(t.financeProfit);
```

`apps/api/src/modules/inter-company/inter-company.service.ts` — 8 instances:
```ts
const shopProfit = Number(t.shopProfit);
const commission = Number(t.commission);
const vatAmount = Number(t.vatAmount);
```

`apps/api/src/modules/sales/sales.service.ts` — 1 instance:
```ts
const costPrice = product ? Number(product.costPrice) : 0;
```

**Required fix**: Use `Prisma.Decimal` for all intermediate arithmetic; serialize with `.toNumber()` only at the JSON response boundary:
```ts
// WRONG
const grossReceivables = Number(hpReceivables._sum.amountDue || 0);

// CORRECT
import { Prisma } from '@prisma/client';
const grossReceivables = new Prisma.Decimal(hpReceivables._sum.amountDue ?? 0);
const netReceivables = grossReceivables.sub(paidOnReceivables);
return { netReceivables: netReceivables.toNumber() }; // only at boundary
```
